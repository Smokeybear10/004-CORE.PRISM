PRISM.
Persistence Reasoning over Idiosyncratic Stock Moves
← back to demo

PRISM is a financial historian. Given a stock that moved unusually big on a given day, it ingests the news, SEC filings, earnings transcripts, and 13F holdings around that move, and asks Claude to attribute the move across five dimensions — each score citing the exact evidence chunks that drove it. Then it decides: is this move structural (lean with it, it'll persist) or transient (fade it, it'll revert)?

§ 01 · What PRISM is

The metaphor: light through a prism

White light enters a prism and refracts into a rainbow of colored beams. PRISM borrows that idea for stocks: a single price move is the white light, and the model decomposes it into five colored beams — each one a dimension of cause, each beam's thickness reflecting how much that dimension drove the move.

§ 02 · How it works

The pipeline, end-to-end

1.Ingest Pull prices, news, transcripts, 10-Ks, 8-Ks, 13Fs from HuggingFace + SEC EDGAR + Finnhub. Cache as parquet on disk.
2.Detect Flag every day where the absolute return exceeded 3× trailing-60d realized vol. About 18 events per ticker over 5 years.
3.Join For each flagged event, pull every evidence chunk dated within the trading-day window — ~50 chunks across 8 sources.
4.Attribute Send the chunks to Claude via a tool_use structured-output call. Claude returns the 5-dimension attribution with chunk-id citations on every score.
5.Coherence Run a second Claude pass (Haiku) that drops any dimension whose rationale doesn't match its citations. Anti-hallucination gate.
6.Decide Classify the move as structural, transient, mixed, or unclear. Convert to a lean / fade / skip verdict via three different strategies.
7.Backtest Apply the verdict to a 5-day forward window. Compare model's PnL against four baselines (naive-lean, always-fade, random, sentiment-only).
8.Cache Memoize every API response by (ticker, date, sources) so the demo never bills the API twice for the same click.

Movement 1 · what comes in
Five external sources fan out into eight evidence streams. Every stream emits typed TextChunks with stable IDs and an enforced publication_date ≤ as_of filter. Cache to disk so re-runs are free.
Movement 2 · what the model does
A flagged price move and its windowed evidence go into a single Anthropic tool_use call. The schema forces five dimensions, each with cited chunks. Validation drops uncited dims; a Haiku second pass audits coherence. The output is a structured Attribution.
Movement 3 · what comes out
Three strategies turn the Attribution into lean / fade / skip. A backtest scores each verdict against four baselines on a 5-day forward return. The full response is cached; FastAPI serves it; the browser renders the prism with every claim backed by a clickable quote.

§ 05 · How to use the demo

An eight-step walkthrough

  1. Pick a ticker.
    Pills at the top of the masthead — currently ABT, ACU, AIR, AMD, APD. Click one. The page populates with that company's 5-year tape and auto-selects the largest-magnitude flagged move.
  2. Pick a move.
    The chart shows the close price as a line, with flagged dots at every day that exceeded the 3-sigma threshold — green dots for rallies, red for sell-offs. Click any dot to focus that move.
  3. Read the verdict at a glance.
    The big italic word in the strip below the chart — Lean. / Fade. / Skip. — is the model's call, color-coded green / red / gray. Underneath it is the strategy that produced it.
  4. Read the attribution (the prism).
    Centerpiece: incoming-event card on the left → triangular prism in the middle → five colored dim cards on the right. Beam thickness shows how much each dimension drove the move. Each dim card shows the top cited quote and the chunk's id.
  5. Ablate sources.
    Bottom-left "Sources" tab has 7 toggle pills. Toggle one off → the model re-runs without that evidence stream → the dim cards and beams update. Watch the attribution shift. This is the demo's "watch the picture sharpen" pitch.
  6. Read the full evidence.
    Same card, switch to the "Evidence" tab. Every cited chunk grouped by dimension, color-coded. Click a dim group to expand → click any citation to see the full quote and the model's reasoning.
  7. Pick a strategy.
    Bottom-right "Strategies" card has three: Fundamental vs Non, Expected vs Realized, Hybrid. Click a row to make it active. The "What the model concluded" panel rewrites in plain English.
  8. Close the loop.
    Below the bottom row: the 5-day forward PnL card. Equity curves show what would have happened if you'd traded each flagged event using the model's verdict — versus four baselines. Then the Eval harness strip at the bottom shows hand-authored ground-truth cases and how often the model got them right.
