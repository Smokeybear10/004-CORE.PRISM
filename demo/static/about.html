PRISM.
Persistence Reasoning over Idiosyncratic Stock Moves
← back to demo

PRISM decomposes individual equity price moves into five attributable dimensions — demand, pricing, competitive dynamics, management credibility, and macro exposure — using qualitative evidence dated within the event window. Every dimensional score is grounded in a specific evidence chunk; uncited scores are dropped at validation. The decomposition produces a structural or transient classification that drives a lean-or-fade verdict, evaluated against four baselines on a five-day forward return.

§ 01 · Method

Dimensional decomposition over qualitative evidence

Sentiment models compress qualitative text into a single scalar. PRISM resists that compression. The same evidence is decomposed across five orthogonal dimensions — chosen to map cleanly onto causal hypotheses about why a stock moved — and scored independently. The vector of dimensional weights, not their sum, is the unit of analysis.

§ 02 · Pipeline

Eight stages from raw filings to forward PnL

1.Ingest Pull prices, news, transcripts, 10-Ks, 8-Ks, 13Fs from HuggingFace + SEC EDGAR + Finnhub. Cache as parquet on disk.
2.Detect Flag every day where absolute return exceeded 3× trailing-60d realized volatility. Approximately 18 events per ticker over five years.
3.Join For each flagged event, pull every evidence chunk dated within the trading-day window — ~50 chunks across 8 sources.
4.Attribute Submit the chunks to a tool_use structured-output call. The model returns the five-dimension attribution with required chunk-id citations on every score.
5.Coherence A second model pass audits whether each dimension's rationale matches its citations and drops any dimension that fails. Hallucination gate.
6.Decide Classify the move as structural, transient, mixed, or unclear. Three strategies map the classification to a lean / fade / skip verdict.
7.Backtest Apply each verdict over a five-day forward window. Compare against four baselines: naive-lean, always-fade, random, and sentiment-only.
8.Cache Memoize every API response by (ticker, date, sources). Re-evaluating the same configuration is free and deterministic.

Stage I · Inputs
Five external sources are normalized into eight typed evidence streams. Every stream emits TextChunks with stable IDs under an enforced publication_date ≤ as_of filter. Disk cache eliminates repeated network calls.
Stage II · Attribution
The flagged move and its windowed evidence enter a single tool_use structured-output call. The schema requires five dimensions and citations on each. Validation drops uncited scores; a coherence pass audits rationale-citation alignment. Output is a typed Attribution.
Stage III · Verdict
Three strategies convert the Attribution to a lean / fade / skip call. The backtest scores each verdict against four baselines on a five-day forward return. Responses are cached; FastAPI serves them; the browser renders the decomposition with every claim linked to its source quote.

§ 05 · Operator manual

Eight steps to a complete decomposition

  1. Select a ticker.
    Masthead pills: ABT, ACU, AIR, AMD, APD. The page loads the five-year tape and selects the largest-magnitude flagged event by default.
  2. Select an event.
    Closing prices appear as a line. Flagged days (those exceeding the 3σ threshold) are marked: green for rallies, red for sell-offs. Clicking a marker focuses the event.
  3. Read the verdict.
    Below the chart: Lean. / Fade. / Skip. — color-coded green, red, or gray, with the active strategy named beneath.
  4. Read the decomposition.
    Center column: event card, decomposition diagram, dimension panels. Beam thickness encodes dimensional weight; each panel surfaces the top cited quote and its chunk identifier.
  5. Restrict the information set.
    "Sources" tab — seven toggles, one per evidence stream. Disabling a source triggers re-attribution under the restricted set; dimension panels and beams update. This is the per-event ablation surface.
  6. Inspect the cited evidence.
    "Evidence" tab — every cited chunk, grouped by dimension. Expanding a group reveals each citation's full quote and the model's rationale.
  7. Switch strategies.
    "Strategies" card — three options: Fundamental vs Non, Expected vs Realized, Hybrid. The conclusion panel restates the verdict under the selected mapping.
  8. Review forward returns.
    The 5-day forward PnL card plots equity curves for the model verdict against four baselines on the flagged event set. The eval-harness strip below reports hand-labeled ground-truth cases and the model's hit rate against them.

§ 06 · Forward returns

Five-day forward PnL versus four baselines

For each flagged event the model's verdict drives a five-day forward sleeve. Four baselines run the same event set under identical notional: naive-lean, always-fade, random, and sentiment-only. The attribution signal claims work only when it dominates every baseline on hit rate and Sharpe. Numbers below are pulled live from the AMD bundle.
