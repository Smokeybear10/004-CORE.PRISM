§ 01

At a glance

The one-screen summary — what PRISM is, the key numbers, and the toggles that matter.

PRISM is a financial historian. Given a 3-sigma stock move, it pulls every news article, SEC filing, earnings transcript, and 13F snapshot dated near the event, then asks Claude to attribute the move across five dimensions — demand · pricing · competitive · management credibility · macro — with each score citing the exact evidence chunks that drove it. The demo lets you toggle which evidence streams the model can see and watch the attribution sharpen.

§ 03

The eight evidence streams

Each chunk is tagged with one SourceType. The user toggles which streams the model is allowed to see.
News news Yahoo Finance + Finnhub headlines & body paragraphs. The immediate narrative around the move. news_AMD_2021-07-28_001
SEC 10-K sec_10k Annual filings. Risk Factors + MD&A only — financial statement digits go to Srilekha's parquet. sec_10k_AMD_2024-02-13_mda_003
SEC 8-K sec_8k Material event filings. Item 2.02 (earnings), 5.02 (exec changes), 7.01 (Reg FD), 8.01 (other). sec_8k_AMD_2024-02-13_item_2.02_001
Earnings transcript earnings_transcript Yahoo Finance call transcripts. Prepared remarks + Q&A separately tagged. earnings_transcript_AMD_2021-04-20_prepared_005
Peer news peer_news Same news parquet, filtered to "family companies" — Apple → Samsung, TSMC, Qualcomm. news_NVDA_2021-07-28_001
Sector news sector_news Sector-wide stories. Catches sector-rotation moves the company-only news misses. news_SECTOR-tech_2021-07-28_001
Macro macro FOMC statements, VIX prints, commodity ticks, geopolitics. Disambiguates idiosyncratic vs. tape moves. macro_FOMC_2021-07-28_statement_001
13F positioning thirteen_f Quarterly institutional holdings. Pre-event positioning signal: crowded longs amplify squeezes / forced-selling. 13f_AMD_2021-06-30_holder-blackrock_001

The repo only ships small artifacts: per-ticker JSON for the demo's first paint, plus the 13F focal chunks file. Everything heavy — HF parquet caches, the API response cache, Claude-derived JSON — stays on each developer's laptop and is regenerable from scratch.

§ 07

Operating modes

One env var picks between cost-permissive (dev) and zero-spend (live demo).

Default · build the cache

BW_USE_LIVE_ATTRIBUTION=1
Use during development and when running prewarm_cache.py. Each MISS bills the API once and saves the answer for next time.

Cache-only · live demo

BW_CACHE_ONLY=1
Hard guarantee no Anthropic call. Use during the actual presentation — any uncached toggle returns 503 instead of silently billing.
